One-Time Setup

ZLib

ZLib is used for data compression within the DevicemEumlator, particularly in the savestate (.dess) files, to save disk space.

Download zlib and expand it into features\zlib. The latest version is recommended: zlib-1.2.3 is known to build and work inside the DeviceEmulator.

LibPNG

LibPNG is used by the skin engine to support .png files.

Download libpng and expand it into features\libpng. The latest version is recommended: libpng 1.2.8 is known to build and work inside the DeviceEmulator.

Building

Open DeviceEmulator.sln in Visual Studio 2005. Use the Build menu to build the entire solution. In a few minutes, it will generate a DeviceEmulator.exe and a 1033\DeviceEmulatorUI.dll. The total disk space required for debug and release builds along with the source tree is around 125mb.

Build Configurations

The Release configuration is the version that ships to customers. Notable settings are:
  • Full C++ optimizations
  • Link-time code generation, to enable cross-module inlining
  • No assertions

The Debug configuration is slower, but easier to debug. Notable settings are:

  • C++ optimizations disabled, to enable source-level debugging
  • Assertions, which trigger a GUI MessageBox containing the assertion text, source file and line number.

Running the Results

The Shared Source DeviceEmulator's command line is identical to the V1 emulator's. To replace the V1 emulator with the Shared Source version, run "DeviceEmulator.exe /register" to execute the emulator's COM self-reg code. This enables the Shared Source emulator to be launched from Visual Studio and from the V1 Device Emulator Manager application, and to participate in ActiveSync-over-DMA.
